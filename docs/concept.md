# Концепция сайта на GitHub Pages: цифровые продукты для двух ниш

**Ниша A:** «Как выйти на маркетплейсы» (WB / Ozon / Я.Маркет / Мегамаркет)
**Ниша B:** «Самозанятые / ИП на услугах» (бьюти, репетиторы, мастера, фрилансеры, консультанты, ремонт и т.д.)

Документ — стратегический разбор. Код не пишем, пока не сойдёмся на варианте.

---

## 1. Что важно понять про GitHub Pages «как платформу для продаж»

GH Pages — **только статика**. Это значит:

- Нет бэкенда → нет «нативной» корзины, нет авторизации, нет приёма платежей собственными руками.
- Любая динамика (оплата, выдача файлов, e‑mail, аналитика воронки) — через внешние сервисы.
- Зато: бесплатно, быстрый CDN, кастомный домен, идеальная карма в SEO (HTML, мгновенные ответы).
- Лимиты: репозиторий до 1 ГБ, ~100 ГБ трафика/мес — для инфо‑сайта более чем достаточно.

Из этого следует, что сайт = **витрина + воронка**, а оплата и доставка живут на стороне:
- **Платёжки (РФ):** ЮKassa, Робокасса, CloudPayments, Tinkoff Касса (через ссылки), Lava.top, Boosty, Telegram Stars (через бота), Digiseller / Glopart / ProductLab.
- **Доставка цифры:** редирект «Спасибо» + ссылка на файл, e‑mail с защищённой ссылкой (Make.com / n8n / Albato + UniSender / Mailganer), личка Telegram‑бота, готовые «комбайны» (Digiseller, GetCourse).
- **Лиды/формы:** Formspree, Getform, Tally, ЯФормы, Telegram‑бот.
- **Аналитика:** Яндекс.Метрика (вебвизор!), GA4, UTM‑разметка, события на кнопках «Купить».
- **Юр.обвязка (обязательно для РФ):** публичная оферта на цифровой продукт, политика конфиденциальности (152‑ФЗ), согласие на рассылку, понятные условия оплаты и возврата.

## 2. Стратегические развилки, которые надо закрыть до архитектуры

| Развилка | Варианты | Что меняется |
|---|---|---|
| **Один бренд или два?** | Один зонтичный (две вертикали внутри) / Два микро‑бренда на одном домене / Два разных сайта | Структура, навигация, SEO, доверие |
| **Личный бренд автора или «как агентство»?** | Эксперт с лицом / Безличный «инфо‑магазин» | Тон, главная, страница «Об авторе», цены |
| **Низкий чек × много SKU** или **высокий чек × мало SKU**? | Магазин чек‑листов/шаблонов по 490–1990 ₽ / Курсы и наставничество от 9 900 ₽ | Тип сайта: витрина vs длинные лендинги |
| **Канал трафика** | SEO / Telegram / Reels‑Shorts / Реклама ВК / Партнёрки | Нужен ли блог, нужен ли бот |
| **Россия‑only или СНГ/мир?** | RU / RU+EN | Платёжки, валюта, договор |
| **Telegram‑first или Web‑first?** | Аудитория живёт в TG / Аудитория ищет в Я.& Google | Где главная воронка |

> **Без ответов на эти 6 пунктов любой выбор архитектуры будет «угадайкой».** Вопросы вынес в финал.

## 3. Пять вариантов архитектуры сайта

Каждый вариант — это **карта страниц + логика воронки + кому подходит + риски**.

### Вариант 1. «Два направления под одним зонтом» (рекомендую как базовый)

**Идея:** одна главная‑хаб с честной развилкой «Куда вы идёте?» → две параллельные вертикали со своими каталогами и лид‑магнитами, общий блог и автор.

```
/                        ← хаб: 2 крупных CTA-блока (Маркетплейсы / Самозанятые)
/marketplace/            ← лендинг направления A
/marketplace/catalog/    ← каталог продуктов A
/marketplace/<slug>/     ← карточка-лендинг каждого продукта A
/services/               ← лендинг направления B
/services/catalog/       ← каталог продуктов B
/services/<slug>/        ← карточка-лендинг каждого продукта B
/blog/                   ← общий блог (SEO)
/blog/<slug>/            ← статья
/about/                  ← автор/эксперт + кейсы + соц.доказательства
/freebies/               ← бесплатные лид-магниты (чек-листы, шаблоны)
/faq/                    ← общие вопросы
/contacts/               ← контакты + реквизиты
/legal/offer/            ← публичная оферта
/legal/privacy/          ← политика конфиденциальности
/thanks/<slug>/          ← страницы «Спасибо» (под каждый продукт, с ссылкой на скачивание)
/404.html, /sitemap.xml, /robots.txt
```

**Воронка:**
1. Холодный трафик → лид‑магнит (бесплатный чек‑лист) → e‑mail/Telegram.
2. Прогрев в рассылке/боте → tripwire (490–990 ₽).
3. Допродажа (order bump на странице оплаты) и upsell (на «Спасибо») в основной продукт.

**Плюсы:**
- Чёткая сегментация трафика → выше конверсия в каждой нише.
- Общий бренд = одна репутация, одна рассылка, одна аналитика.
- SEO‑бонус: блог тянет обе вертикали.

**Минусы:**
- Главная сложнее — нужно за 5 секунд дать понять, что здесь две темы.
- Чуть больше страниц на поддержку.

**Кому:** если у вас **один эксперт/команда**, который уверенно говорит на обе темы, и средний чек 990–9 900 ₽.

---

### Вариант 2. «Каталог‑витрина в стиле Gumroad/ProductLab»

**Идея:** сайт = маленький магазин цифровых товаров с фильтрами по категории / формату / цене. Карточки короткие, но с яркими обложками. Покупка в 2 клика.

```
/                  ← hero + сетка популярных продуктов + соц.доказательства
/products/         ← каталог с фильтрами: Ниша (МП/Услуги), Формат (PDF/Notion/Excel/Видео), Цена
/products/<slug>/  ← короткая карточка: обложка + 5 буллетов + цена + «Купить»
/bundles/          ← наборы (со скидкой) — основной драйвер AOV
/blog/             ← опционально (можно вынести в TG)
/about/, /faq/, /contacts/, /legal/...
```

**Плюсы:**
- Идеально для **много мелких SKU** (490–2 990 ₽): чек‑листы, шаблоны договоров, таблицы юнит‑экономики, скрипты переписки с клиентом, шаблоны Reels.
- Быстро добавлять новые продукты (один MD/MDX‑файл = новая карточка).
- Высокий AOV за счёт бандлов.

**Минусы:**
- Слабее работает для дорогих продуктов (нужен длинный лендинг).
- Без сильного личного бренда выглядит «обезличенно».

**Кому:** если планируете **30+ цифровых товаров**, низкий–средний чек, много трафика из Pinterest/Reels.

---

### Вариант 3. «Контент‑маркетинг: блог‑сайт + встроенные продукты»

**Идея:** сайт построен вокруг SEO‑блога. Каждая статья — это ответ на конкретный поисковый запрос («как вывести бьюти‑мастера в топ Я.Услуг», «как открыть магазин на WB в 2026»), внутри статьи — встроенный CTA на платный продукт.

```
/                ← подборки лучших статей + 3 главных продукта
/blog/           ← рубрики: Маркетплейсы, Самозанятые, Налоги, Маркетинг
/blog/<slug>/    ← статья + блок «Скачать готовый шаблон» (платно)
/shop/           ← каталог продуктов (как в варианте 2, но проще)
/shop/<slug>/    ← карточка
/freebies/, /about/, /faq/, /legal/...
```

**Плюсы:**
- Бесплатный органический трафик из Я./Google в долгую.
- Каждая статья = отдельная точка входа в воронку.
- Хорошо ложится на Astro/Hugo с MDX.

**Минусы:**
- Окупается через 4–9 мес; если нет ресурса писать 2–4 статьи/нед — не работает.
- Нужен редактор/копирайтер или дисциплина автора.

**Кому:** если готовы **играть в долгую SEO** и есть экспертиза, которой не жалко делиться текстом.

---

### Вариант 4. «Воронка из лендингов» (классика русского инфобиза)

**Идея:** на каждый продукт — отдельный длинный продающий лендинг (5–15 экранов). Корзины как таковой нет, есть «Купить» → внешняя страница оплаты → «Спасибо» с upsell.

```
/                          ← навигация по направлениям + топ-3 продукта
/m/<slug>-marketplace/     ← длинный лендинг продукта A1
/m/<slug>-marketplace/     ← длинный лендинг продукта A2
/s/<slug>-services/        ← длинный лендинг продукта B1
...
/upsell/<slug>/            ← апсейл-страница после оплаты
/thanks/<slug>/            ← выдача + следующий оффер
/about/, /blog/, /freebies/, /legal/...
```

**Плюсы:**
- Самая высокая конверсия на платном трафике (ВК/Я.Директ/Telegram Ads).
- Можно сильно поднять LTV через order bump + upsell + downsell.

**Минусы:**
- Каждый лендинг = много работы (тексты, дизайн, кейсы, видео).
- Быстро добавлять новые SKU не получится.
- Плохо «дружит» с SEO‑витриной — это другой жанр.

**Кому:** если **2–5 флагманских продуктов** с чеком 5 000–50 000 ₽ и закупаете трафик.

---

### Вариант 5. «Гибрид: сайт‑витрина + Telegram‑бот = ядро бизнеса»

**Идея:** сайт нужен для SEO, доверия и презентации, но **главная воронка живёт в Telegram‑боте**: лид‑магнит, прогрев, оплата (Lava.top / ЮKassa в боте / Telegram Stars), доставка цифры прямо в чат.

```
/                ← короткая презентация + большой CTA «Открыть бота»
/products/       ← каталог-превью (ссылки → бот)
/products/<slug>/← карточка-превью с кнопкой «Купить в боте»
/blog/           ← опционально
/about/, /faq/, /legal/, /freebies/ → все CTA ведут в бота
```

**Плюсы:**
- Аудитория «Самозанятые / WB‑продавцы» очень telegram‑центрична — конверсия в подписку выше, чем в e‑mail.
- Доставка цифры мгновенно, без писем в спам.
- Сегментация подписчиков и автопрогрев — встроены в бота.

**Минусы:**
- Сайт в этой схеме — вторичен; если хочется «настоящий магазин с поиском», вариант не подходит.
- Завязка на Telegram (риски блокировок/ограничений).

**Кому:** если уже есть/хочется Telegram‑канал и аудитория.

## 4. Какой вариант выбрать — короткая матрица

| Сценарий | Рекомендую |
|---|---|
| 5–15 продуктов, средний чек, хочу растить и SEO, и платный трафик | **Вариант 1** |
| 30+ мелких товаров, упор на скорость добавления | **Вариант 2** |
| Готов писать много, бюджета на рекламу нет | **Вариант 3** |
| 2–5 флагманов, льёте платный трафик | **Вариант 4** |
| Аудитория в Telegram, хочется автоматизации | **Вариант 5** |

> Самые жизнеспособные для старта — **1 и 5** (или их комбинация: 1 как «лицо», 5 как «движок»).

## 5. Технологический стек (под GH Pages)

**Генератор сайта** (по убыванию моей рекомендации):
1. **Astro** — лучший выбор для контентно‑продуктового сайта: коллекции контента, MDX, минимум JS, встроенный SEO, «острова» только там, где нужно.
2. **Hugo** — самые быстрые сборки, отличные темы, но шаблоны на Go менее приятны.
3. **Jekyll** — нативный для GH Pages, но тяжёлый для масштабирования.
4. **Next.js (static export)** — если уверенно владеете React и хотите сложные интерактивы.
5. **11ty** — гибко, легко, но мало готовых тем.

**Стилизация:** Tailwind CSS — стандарт де‑факто для лендингов.
**Контент:** MDX/Markdown в репозитории, **CMS опционально** — Decap CMS / TinaCloud / Sveltia (бесплатно, прямо поверх GH).
**Поиск по сайту:** Pagefind (статический индекс, отлично для GH Pages).
**Формы:** Formspree / Getform / Tally / прямой POST в Telegram‑бота.
**Платежи (РФ):**
- **Lava.top** — проще всего для самозанятого/ИП, поддерживает оплату картой/СБП/Telegram Stars и мгновенную выдачу файла после оплаты.
- **ЮKassa** — если уже есть ИП и хочется максимально «по‑белому».
- **Digiseller** — готовый магазин цифровых товаров; можно встроить кнопки «Купить» на свой сайт.
- **Boosty** — для подписочных моделей и разовых покупок.
**Доставка цифры:** ссылка на «Спасибо» + e‑mail с защищённой ссылкой (signed URL из S3‑совместимого Selectel/VK Cloud) или сразу выдача в Telegram‑боте.
**Аналитика:** Я.Метрика (вебвизор обязателен) + GA4, события на «Купить», «Скачать», UTM на всех CTA.
**Деплой:** GitHub Actions → ветка `gh-pages` → кастомный домен с SSL.

## 7. Юридический минимум (РФ) — нельзя забыть

- **Публичная оферта** на каждый цифровой продукт (момент заключения договора = оплата).
- **Политика обработки персональных данных** + согласие при сборе e‑mail (152‑ФЗ).
- **Согласие на рекламную рассылку** (отдельный чек‑бокс, не предзаполненный).
- **Подтверждение оплаты и доступа:** в оферте должны быть понятные правила доставки цифрового продукта и возврата.
- **Контакт продавца** на сайте и в юридических документах.
- **Возврат:** для цифровых товаров — на ваше усмотрение, но условия должны быть написаны в оферте.

### Вопросы, на которые нужны ответы

1. **Ниша A и B — обе ваши экспертизы или одну будете аутсорсить?** (от этого зависит, один бренд или два)
2. **Сколько продуктов планируете на старте и какой средний чек?** (5 продуктов по 9 900 ₽ — это вариант 1/4; 30 продуктов по 990 ₽ — это вариант 2)
3. **Откуда будет идти трафик в первые 3 месяца?** (SEO / Telegram / Reels / Я.Директ / партнёрки / нет идей)
4. **Есть ли у вас ИП или самозанятость уже?** (от этого зависит выбор платёжки)
5. **Telegram‑first или Web‑first аудитория?** (нужен ли бот как ядро воронки)
6. **Нужен ли блог?** (готовы писать 4+ статьи/мес или нет)
7. **Личный бренд (с лицом автора) или безличный «инфо‑магазин»?**
8. **Дизайн:** хочется минимализм/«нотион»‑стиль / яркий «продающий» / «премиум»?
9. **Языки:** только RU или RU+EN на будущее?
10. **Бюджет на инструменты в месяц** (домены, рассылка, конструкторы — порядок суммы)?

После ответов сделаю **финальный план с конкретной картой страниц, текстовым прототипом главной и пошаговой дорожной картой запуска**.
